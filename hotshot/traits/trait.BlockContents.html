Expand description

Abstraction over the contents of a block

This trait encapsulates the behaviors that a block must have in order to be used by consensus:

  • Must have a predefined error type (BlockContents::Error)
  • Must have a transaction type that can be compared for equality, serialized and serialized, sent between threads, and can have a hash produced of it (hash_transaction)
  • Must be able to be produced incrementally by appending transactions (add_transaction_raw)
  • Must be hashable (hash)
