Expand description

Abstraction over the state that blocks modify

This trait represents the behaviors that the ‘global’ ledger state must have:

  • A defined error type (Error)
  • The type of block that modifies this type of state (Block)
  • A method to get a template (empty) next block from the current state (next_block)
  • The ability to validate that a block is actually a valid extension of this state (validate_block)
  • The ability to produce a new state, with the modifications from the block applied (append)
