Expand description

Trait for a stateful event handler

The PhaseLock instance will keep around the provided value of this type, and call the notify method every time a series of blocks are committed.

A do-nothing implementation (Stateless) is provided, as a convince for implementations that do not need this functionality.

The PhaseLock implementation will call this method, with the series of blocks and states that are being committed, whenever a commit action takes place.

The provided states and blocks are guaranteed to be in ascending order of age (newest to oldest).

This functionality is discretionary, and whatever behavior is implemented in this method should not impact consensus logic.
