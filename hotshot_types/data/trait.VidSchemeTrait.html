fn verify_share( &self, share: &Self::StorageShare, common: &Self::StorageCommon ) -> Result<Result<(), ()>, VidError>

Verify a share. Used by both storage node and retrieval client. Why is return type a nested Result? See https://sled.rs/errors Returns:

  • VidResult::Err in case of actual error
  • VidResult::Ok(Result::Err) if verification fails
  • VidResult::Ok(Result::Ok) if verification succeeds

fn recover_payload( &self, shares: &[Self::StorageShare], common: &Self::StorageCommon ) -> Result<Vec<u8, Global>, VidError>

Recover payload from shares. Do not verify shares or check recovered payload against anything.
